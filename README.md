# CV + IK Pipeline для SO-101

Pick-and-place pipeline для робота SO-101: детекция куба камерами, локализация через подобие треугольников, захват и перенос.

## Путь решения

1. **SmolVLA (end-to-end)** - начали с обучения VLA-модели. Собрали датасет в MuJoCo, обучили ~100k шагов на симуляции, дообучили ~10k на 15 реальных эпизодах. Даже после такого объёма обучения рука билась в конвульсиях - модель не давала стабильного управления. Решили перейти к классическому CV.
2. **HSV-фильтр** - первая попытка обойтись без нейросетей. Цветовая сегментация + контуры. Слишком хрупко при изменении освещения.
3. **YOLO-World** - zero-shot детекция. Заработало сразу и идеально. Классы: `["cube", "wooden block", "small box", "orange ball", "ball"]`.
4. **Геометрическая локализация** - пробовали ray-plane intersection, стерео-пару, гомографию. Все давали ошибки из-за неточных параметров камер.
5. **Подобие треугольников** - `k = h_px * D`. Одна калибровочная точка на камеру. CAM1 (фронтальная) даёт X, CAM2 (боковая) даёт Y. Просто, но всё равно не заработало - требует тчательной отладки.
6. **FK/IK** - извлекли кинематику из MuJoCo XML. Направления верные, но абсолютные координаты не совпадают с физикой (~70-90% точности). Для грубого позиционирования достаточно.
7. **Кинестетическое обучение** - записали ключевые позы (home, above_pick, pick, above_place, place) через телеоперацию. Надёжнее чем чистый IK.

## Зависимость `lerobot`

Для управления реальным SO-101 нужен `lerobot` (в проекте он используется через `robot_controller.py`).

Установка:

1. Клонировать репозиторий `lerobot`:
   - `git clone https://github.com/huggingface/lerobot.git`
2. Добавить путь к `lerobot/src` в `PYTHONPATH` (или поправить путь в коде):
   - в `pick_and_place.py` есть явный `sys.path.insert(..., ".../lerobot/src")`
   - пример пути: `~/projects/lerobot/src` (подставьте ваш реальный путь до клона)
3. Установить зависимости `lerobot` по их инструкции (из `lerobot`-репозитория), чтобы импорт работал.

Как используется в `pick_and_place.py`:

- через `SO101Controller` импортируются модули `lerobot.robots.so_follower` для подключения к роботу;
- при создании контроллера `calibration_dir` задаётся вручную (в коде сейчас передаётся явный путь);
- если у вас стандартный кэш Hugging Face, обычно это `~/.cache/huggingface/lerobot/calibration/robots/so_follower`, иначе укажите ваш фактический путь;
- чтобы проверить фактический путь, откройте `~/.cache/huggingface/lerobot/calibration/robots/` и используйте существующую директорию вашего робота;
- без доступного `lerobot` и корректной калибровки скрипт не сможет подключиться к роботу.

## Визуальный серворегулятор (нереализованная идея)

Думаем что идея гениальная, но придумали под конец.

**Суть:** вместо вычисления абсолютных координат куба и решения обратной кинематики - использовать камеры как датчики обратной связи. Куб, стоящий в одном и том же реальном месте, всегда выглядит одинаково на обеих камерах. Значит:

1. Ставим куб в "заветное место" - запоминаем `h_px` с двух камер (`capture_target.py`)
2. Клешня толкает куб по столу, пока `h_px` на обеих камерах не совпадёт с целевым
3. Ошибка CAM1 → двигать по X, ошибка CAM2 → двигать по Y (P-регулятор)
4. Совпало → запускаем заранее записанный сценарий подъёма из этой точки

Никакого FK/IK для позиционирования. Никакой калибровки координат камера→робот. Только пиксели и пропорциональный регулятор. Робот сам приводит куб туда, откуда умеет его поднимать.

Скрипты `capture_target.py` и `servo_push.py` - это реализация шагов 1-3. Шаг 4 (сценарий подъёма) - в `poses.json` + `pick_and_place.py`.
